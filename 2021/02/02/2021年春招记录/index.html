      • 虚拟内存
        • 虚拟内存是计算机系统内存管理的一种技术。它使得应用程序认为它拥有连续可用的内存（一个连续完整的地址空间），而实际上，它通常是被分隔成多个物理内存碎片，还有部分暂时存储在外部磁盘存储器上，在需要时进行数据交换
        • 物理内存的容量是固定的，但是寻址空间却取决于cpu地址线条数，如32位机，则寻址空间为2^32 = 4G，所以最大支只持4G的寻址空间，即使插了8G的内存条也只能使用4G内存
        • 操作系统向进程描述了一个完整的连续的虚拟地址空间供进程使用，但是在物理内存中进程数据的存储采用离散式存储(提高内存利用率)，但是其实虚拟内存和物理内存之间的关系并不像上图中那样直接，其中还需要使用页表映射虚拟地址与物理地址的映射关系，并且通过页表实现内存访问控制。
        • 页表是一种特殊的数据结构，存放着各个虚拟页的状态，是否映射，是否缓存.。进程要知道哪些内存地址上的数据在物理内存上，哪些不在，还有在物理内存上的哪里，这就需要用页表来记录

- 解释引用和指针的区别

  - > - 指针所指向的内存空间在程序运行过程中可以改变，而引用所绑定的对象一旦绑定就不能改变。（是否可变）
    > - 指针本身在内存中占有内存空间，引用相当于变量的别名，在内存中不占内存空间。（是否占内存）
    > - 指针可以为空，但是引用必须绑定对象。（是否可为空）
    > - 指针可以有多级，但是引用只能一级。（是否能为多级）

     * 每个数组一个指针，碰到全部相等的数字就加入结果

    class Solution {
    public:
        vector<int> arraysIntersection(vector<int>& arr1, vector<int>& arr2, vector<int>& arr3) {
            vector<int> res;
            int minLength = std::min(arr1.size(), std::min(arr2.size(), arr3.size()));
            int ptr1 = 0, ptr2 = 0, ptr3 = 0;
            while (ptr1<minLength && ptr2<minLength && ptr3<minLength) {
                if (arr1[ptr1]==arr2[ptr2] && arr2[ptr2]==arr3[ptr3]) {

                    ++ptr1;
                    ++ptr2;
                    ++ptr3;

                    ++ptr1;
                } else if (arr2[ptr2]<=arr1[ptr1] && arr2[ptr2]<=arr3[ptr3]) {
                    ++ptr2;
                } else if (arr3[ptr3]<=arr1[ptr1] && arr3[ptr3]<=arr2[ptr2]) {
                    ++ptr3;
                }
            }
            return res;
        }

- TCP当中为什么握手比挥手多一次

  - > TCP协议是一种面向连接的、可靠的、基于字节流的传输层通信协议，采用全双工通信。
  >
    > **那为什么需要三次握手呢？**请看如下的过程：
  >
    > 1. A向B发起建立连接请求：A——>B；
  > 2. B收到A的发送信号，并且向A发送确认信息：B——>A；
    > 3. A收到B的确认信号，并向B发送确认信号：A——>B。
  >
    > 三次握手大概就是这么个过程。
  > 通过第一次握手，B知道A能够发送数据。通过第二次握手，A知道B能发送数据。结合第一次握手和第二次握手，A知道B能接收数据。结合第三次握手，B知道A能够接收数据。
    >
    > 至此，完成了握手过程，A知道B能收能发，B知道A能收能发，通信连接至此建立。三次连接是保证可靠的最小握手次数，再多次握手也不能提高通信成功的概率，反而浪费资源。
    >
    > **那为什么需要四次挥手呢？**请看如下过程：
    >
    > 1. A向B发起请求，表示A没有数据要发送了：A——>B；
    > 2. B向A发送信号，确认A的断开请求请求：B——>A；
    > 3. B向A发送信号，请求断开连接，表示B没有数据要发送了：B——>A；
    > 4. A向B发送确认信号，同意断开：A——>B。
    >
    > B收到确认信号，断开连接，而A在一段时间内没收到B的信号，表明B已经断开了，于是A也断开了连接(这一段时间就是**TIME_WAIT**)。至此，完成挥手过程。
    >
    > 为什么2、3次挥手不能合在一次挥手中？那是因为此时A虽然不再发送数据了，但是还可以接收数据，B可能还有数据要发送给A，所以两次挥手不能合并为一次。
    >
    > 挥手次数比握手多一次，是因为握手过程，通信只需要处理**连接**。而挥手过程，通信需要处理**数据+连接**。

- 一台服务器当中有大量TIMA_WAIT，为什么

  - 查看处于Time_wait的套接字数量`netstat|grep -o TIME_WAIT|wc -l`
  - 产生大量Time_wait的根本原因是频繁创建、断开TCP连接
  - 解决方案是把频繁创建的短连接改为长连接
    - 后台业务服务器，通常需要调用redis、mysql以及其他http服务和grpc服务。在服务相互调用中，如果使用的是短连接，高并发时就会产生大量TIME_WAIT。redis等客户端会有连接池，我们要做的是设置好相关的连接服务参数，一般会有连接数、连接重用时间、连接空闲数等。所以在应用中通过设置合理的连接池参数可以避免TIME_WAIT状态过多的问题。
    - 网络情况不好时，如果主动方无TIME_WAIT等待，关闭前个连接后，主动方与被动方又建立起新的TCP连接，这时被动方重传或延时过来的FIN包过来后会直接影响新的TCP连接。同样网络情况不好并且无TIME_WAIT等待，关闭连接后无新连接，当接收到被动方重传或延迟的FIN包后，会给被动方回一个RST包，可能会影响被动方其它的服务连接。(这种情况不太懂)

      • map和unordered_map的底层实现

        • 实现方式：unordered_map内部是一个hash_table一般是由一个大vector元素节点挂接链表来解决冲突。map的内部实现是二叉平衡树(红黑树)，可在O(logN) 时间内完成查找、增加、删除操作
        • 运行效率方面：unordered_map查找更快，map插入和删除更快
        • 占用内存方面：map内存占用略低，unordered_map内存占用略高，而且是线性成比例的
        • 使用选择方面：需要无序容器，快速查找删除，不担心略高的内存时用unordered_map；需要有序容器插入和删除操作居多，内存在意的时候用map

      • set和unordered_set的底层实现

        • 实现方式：set通过红黑树实现，插入、查找、删除的平均复杂度都是O(logn)。unordered_set通过哈希表实现，容器的插入、查找、删除，正常情况下复杂度是O(1)，最坏情况下时间复杂度是O(n)

          • 可以在构造函数中精简代码

          • ////< @note std::move
void construct_foo_by(Foo&& rhs) { 
   Foo foo(std::move(rhs));
}

void construct_foo_by(const Foo& rhs) { 
    Foo foo(rhs);
}

int main(int argc, char const *argv[]) {
    Foo foo(10);
    construct_foo_by(foo);      // 调用 ctor
    construct_foo_by(Foo{100}); // 调用 mtor
    return 0;
}

////< @note std::forward
template<typename T> 
void create_foo_by(T&& rhs) { 
    Foo foo(std::forward<T&&>(rhs));
}

int main(int argc, char const *argv[]) {
    Foo foo(10);

    create_foo_by(foo);   // ctor
    create_foo_by(Foo{100}); // mtor

    return 0;
}

学习路线

插一张书本的照片

C++基础

  • 《C++ Primer（第五版）》
  • 《Effective C++》侯捷著
  • 《深度探索C++对象模型》侯捷译
  • 《STL源码剖析》侯捷著

上述这几本书可以说是C++学习必读书目，面试问C++基础逃不出这几本书，建议认真阅读一遍。设计模式方面掌握工厂模式、观察者模式和单例模式能够应付大部分面试了。

网络编程

  • 《TCP/IP网络编程》尹圣雨著
  • 《Linux高性能服务器编程》游双著
  • 《Linux多线程服务端编程》陈硕著

上述三本书按照难度和学习顺序排列，第一本书是一个韩国人写的，很基础适合入门，后面两本书是C++服务器开发的进阶书籍

操作系统

  • 《深入理解计算机系统》CSAPP
  • 《现代操作系统（第二版）》

操作系统学习书籍不用多说CSAPP永远的经典

数据库

  • 《MySQL必知必会》
  • B站一天学会MySQL

第一本书是由浅入深的学习书籍，如果嫌看书太慢，可以直接去b站根据关键词“一天学会MySQL”进行搜索，几个视频都挺不错的

数据结构

  • 《剑指offer》
  • 力扣前300题

数据结构方面《剑指offer》必刷，力扣前300题尽量也全部过一遍，面试手撕很多原题都出自这里。力扣官网上有一个学习专区，很多leetbook值得一看，会员的话可以淘宝上买一个，价格很便宜。刷题语言建议主力开发语言是什么就用什么，很多人会说用python刷题会快，我也尝试过，但刷题更讲究思路而不是语言，总体试下来C++和python差不多，反而在刷题过程中我能够更进一步熟悉C++的语言特性。
